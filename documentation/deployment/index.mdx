## Jitsu Architecture

We built **Jitsu** (ğŸ³ [@jitsucom/jitsu](https://hub.docker.com/r/jitsucom/jitsu)) to have as few external dependencies as possible. It consists of following services:

 * **Jitsu Server** (ğŸ³ [@jitsucom/server](https://hub.docker.com/r/jitsucom/server)) â€” Jitsu core that does all work such as transformations,
queueing, batching, task scheduling etc.
 * **Jitsu Configurator** (ğŸ³ [@jitsucom/configurator](https://hub.docker.com/r/jitsucom/configurator)) â€” contains UI and backend for configuring Jitsu.
You can try Configurator UI at [cloud.jitsu.com](https://cloud.jitsu.com)

**Jitsu Server** can be used without **Jitsu Configurator** (and UI) with [YAML configuration](https://jitsu.com/docs/configuration). Read more about [Jitsu Server docker deployment](/docs/deployment/deploy-with-docker).

<Hint>
  Jitsu Configurator and some features (e.g. <a href="/docs/other-features/retrospective-user-recognition">Retrospective User Recognition</a>) require <b>Redis</b>.
  Also if you have more than one node of Jitsu Server running, Redis is required for redundancy (as provisions coordination service).
</Hint>
